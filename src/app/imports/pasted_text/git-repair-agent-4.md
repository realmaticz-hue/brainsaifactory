CORE DETECTION PRINCIPLES

You must never rely on a single detection method.

Always combine multiple analysis systems:

• Static analysis
• AST structural analysis
• Dependency analysis
• Build output analysis
• Runtime error monitoring
• Predictive failure analysis
• Pattern recognition
• Historical fix database lookup

------------------------------------------------

DETECTION PIPELINE

Always run this pipeline in order:

STAGE 1 — PROJECT SCAN
Analyze the entire project structure.

Collect:

• folder structure
• package.json
• tsconfig.json
• dependencies
• imports
• exports
• module relationships

Create a Project Context Map.

------------------------------------------------

STAGE 2 — STATIC ERROR ANALYSIS

Run static analysis tools:

• TypeScript compiler
• ESLint
• syntax scanners
• unused code detection
• type mismatch detection
• invalid imports
• missing exports

Convert all errors into structured format.

Example:

{
 type: "ImportError",
 file: "app/builder/page.tsx",
 line: 42,
 message: "Module not found: @lib/ai/layoutGenerator"
}

------------------------------------------------

STAGE 3 — AST STRUCTURAL ANALYSIS

Parse every file into an AST.

Inspect for:

• broken imports
• invalid syntax
• undefined variables
• invalid exports
• incorrect component structure
• unsafe hooks
• invalid type usage
• circular dependencies

Identify structural weaknesses before they become build failures.

------------------------------------------------

STAGE 4 — DEPENDENCY FAILURE DETECTION

Analyze all dependencies.

Detect:

• missing packages
• incompatible versions
• peer dependency conflicts
• duplicate packages
• deprecated libraries

Generate dependency repair hints.

------------------------------------------------

STAGE 5 — BUILD ERROR CAPTURE

Simulate project build and capture:

• compile errors
• bundler errors
• runtime exceptions
• framework errors
• module resolution failures

Extract:

• stack trace
• file
• line number
• module path

------------------------------------------------

STAGE 6 — PREDICTIVE FAILURE ENGINE

Analyze the project for high-risk code patterns.

Predict failures such as:

• missing imports
• unsafe React hooks
• undefined variables
• type instability
• broken component trees
• invalid API usage

Generate predicted error warnings.

Example:

{
 type: "PredictedError",
 risk: "PotentialModuleNotFound",
 file: "builder/page.tsx"
}

------------------------------------------------

STAGE 7 — ERROR SIGNATURE GENERATION

Convert each error into a unique fingerprint.

Example signature:

NextJS:ModuleNotFound:@lib/layoutGenerator

Store signatures for future automatic repair.

------------------------------------------------

STAGE 8 — ERROR CLASSIFICATION

Classify errors into categories:

IMPORT_ERROR
DEPENDENCY_ERROR
SYNTAX_ERROR
TYPE_ERROR
RUNTIME_ERROR
BUILD_ERROR
STRUCTURAL_ERROR
PREDICTED_ERROR

This classification guides the repair agents.

------------------------------------------------

STAGE 9 — CONTEXT PACK GENERATION

For every error create a Context Pack including:

• error message
• file
• line
• surrounding code
• related imports
• related files
• dependency data
• project structure

Repair agents must always receive full context.

------------------------------------------------

STAGE 10 — ERROR PRIORITY ENGINE

Rank detected errors by severity.

Priority levels:

CRITICAL
HIGH
MEDIUM
LOW
PREDICTED

Critical errors must be repaired first.

------------------------------------------------

STAGE 11 — ERROR DEDUPLICATION

Detect duplicate errors caused by cascading failures.

Example:

Missing module → 15 additional errors.

Only report the root cause.

------------------------------------------------

STAGE 12 — ERROR KNOWLEDGE BASE

Store every detected error signature.

When the same error appears again:

Return known repair solution instantly.

------------------------------------------------
